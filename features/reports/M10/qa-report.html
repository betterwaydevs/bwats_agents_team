Bug Fix: first_notnull Filter Round 2

Root Cause Found & Fixed: The XanoScript first_notnull pipe filter throws "Not numeric." when applied to text fields on the development data source. The filter internally attempts a numeric conversion that fails on string values.

Fix: Replaced all first_notnull usages with JavaScript || null coalescing inside api.lambda blocks. Functionally identical, but avoids Xano's internal type coercion bug.

Occurrences fixed:

  • $person.first_name|first_notnull:"" → p.first_name || ''
  • $person.last_name|first_notnull:"" → p.last_name || ''
  • $person.headline_role|first_notnull:"" → p.headline_role || ''
  • $project.name|first_notnull:"" → $var.project.name || ''

Deployed via: MCP updateAPI to workspace 6, API group 1470, endpoint 43105, with publish: true.
