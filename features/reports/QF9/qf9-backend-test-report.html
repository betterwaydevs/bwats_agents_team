Observations & Notes

Intermittent Latency Spikes: 2 out of 9 successful calls experienced extreme response times (31s, 96s). These are NOT caused by the endpoint code (which has no blocking operations). They appear correlated with process_pending_prospects scheduled task activity, possibly due to database row locks during Add_prospect processing. This is a platform-level concern, not a QF9 regression.
Async Pipeline Backlog: There are 82 pending records as of testing. At ~1 record/minute processing rate, the backlog will take ~80-160 minutes to clear. This is pre-existing — QF9 did not create the backlog. Consider increasing processing throughput in a future ticket.
Duplicate Detection: Calls without profile_id consistently hit a "Duplicate record detected" error (database unique constraint). The extension always sends profile_id, so this is not a production issue, but the endpoint could handle this edge case more gracefully.
