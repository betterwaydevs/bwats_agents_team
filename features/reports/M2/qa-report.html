Bugs Found (non-blocking)

BUG-1: Breakdown field name mismatch (backend ↔ frontend)

The API returns email_account_id/email_address for account breakdown and project_id/project_title for project breakdown. The frontend expects group_id and group_label. The breakdown table's Name column will render empty.

Fix: Either normalize field names in the backend to use group_id/group_label, or add a mapping layer in the frontend service (emailStatsApi.ts).

BUG-2: Account filter not applied to breakdown query

When filtering by email_account_id=22, the summary correctly shows 13 sent (matching account 22), but the breakdown still returns all 9 accounts instead of just account 22.

Fix: Apply the email_account_id filter to the breakdown aggregation query in the backend.
