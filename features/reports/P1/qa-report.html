What Was Fixed

The prospects grid in the project view would freeze (become unresponsive) after stage moves and task completions. The root causes were:

  1. Table unmount/remount on refetch — Each data refresh caused the entire table to unmount and remount, losing interactive state.
  2. N+1 API calls — After each page load, the code fired 50+ individual API calls (one per person) via Promise.all(items.map(...)), causing cascading refetches.
  3. No staleTime — Queries refetched too aggressively, compounding the unmount/remount issue.

Fix Applied

  • Added placeholderData: keepPreviousData to prevent table unmount during refetch
  • Removed N+1 Promise.all(items.map(...)) pattern — now uses embedded data from the API response
  • Added staleTime: 30_000 (30 seconds) to reduce unnecessary refetches
