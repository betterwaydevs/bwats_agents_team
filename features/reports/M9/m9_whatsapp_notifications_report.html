Issues & Notes

Performance concern: The GET /messaging/whatsapp_unread endpoint can be slow (~30-60s) when doing phone number matching via parsed_candidate.phone_number includes. This is a full-text search on unmatched contacts. Once a contact is matched (person_id backfilled), subsequent requests skip the search. Consider adding a phone index or caching.
Slug typo: The person_slug for candidate 9547 is generated as pablo-velsquez-9547 (missing 'a' in Velasquez). This comes from the source data in parsed_candidate, not from the slug generation logic.
