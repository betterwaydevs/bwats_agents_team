Failures

Test 9: Search timeout on large project

Issue: POST /association/search returns 502 when searching across 7,128 prospects in project 5

Root cause: ElasticSearch query timeout on large datasets

Workaround: Filter by stage first to reduce dataset size

Severity: Low — affects only very large projects

Test 14: Person associations endpoint 404

Issue: GET /association/person/prospect/{esId} returns 404

Possible causes: Endpoint may require different URL pattern, or user role permissions

Severity: Low — person details available via direct detail endpoints

Key Findings

1. People listing names are blank: The _person join returns null names for many prospects imported via LinkedIn. Skill must fetch full detail from prospect/candidate endpoint.
2. Must route by person_type: Calling candidate endpoint with a prospect's ES ID returns all nulls. Always check person_type from the association first.
3. Dual auth headers required: Both Authorization and X-Xano-Authorization must be sent — different endpoints check different ones.
4. Pagination shape is consistent: All paginated endpoints return {items, itemsTotal, itemsReceived, curPage, nextPage} as documented.

Conclusion

The /ats skill's read operations are functional for all core workflows: dashboard display, project listing, pipeline visualization, stage counts, people listing (with detail fetch for names), and search (on reasonably-sized projects). All findings have been incorporated as gotchas in the skill definition.
