Additional Data

The new endpoint includes a failed count (currently 0) which the old endpoint didn't track separately. This provides better visibility into the prospect parsing pipeline.

Dropped fields (intentional)

The new endpoint drops newest_prospect_date (MAX(created_at)) from the old endpoint. This was an intentional design decision for performance — computing MAX per group requires examining every row's timestamp. The date was not critical for the count display use case.
