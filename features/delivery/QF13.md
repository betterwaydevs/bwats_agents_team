# QF13 — Pending Prospect Count Performance — Delivery Report

> **Status**: deployed-v1 | **Date**: 2026-03-03

## Summary

Created a new `prospect_parse_status_counts` GET endpoint that replaces the slow `unparsed_count` endpoint. The new endpoint uses three separate indexed `db.query` COUNT operations instead of a raw SQL query with COALESCE/NULLIF/GROUP BY, enabling index-only scans on the existing btree index on `parse_status`.

### New File
- `apis/prospects/prospect_parse_status_counts_GET.xs` — New fast count endpoint

### Old Endpoint (to be deprecated)
- `apis/prospects/16900_unparsed_count_GET.xs` — Existing slow endpoint using `db.direct_query` with COALESCE
