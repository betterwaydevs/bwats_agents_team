### Findings

#### [MEDIUM]: Unvalidated download URL protocol — BwatsDownloads.tsx:29
**File**: `src/apps/bwats/pages/BwatsDownloads.tsx:29`
**Issue**: `window.open(tool.download_url, '_blank')` opens whatever URL the API returns without protocol validation. If a malicious URL (e.g., `javascript:`, `data:`, or a phishing domain) were injected into the `downloadable_tools` table, clicking Download would open it.
**Impact**: Potential phishing redirect or (in older browsers) script execution via `javascript:` protocol URLs.
**Mitigating factors**: (1) Only admin users can write to `downloadable_tools` via `POST /api/tools/upload`, so the attack surface requires a compromised admin account or direct DB access. (2) Modern browsers block `javascript:` URLs in `window.open` with `_blank`. (3) This is an internal-only tool used by a small team.
**Recommendation**: Add a protocol check before opening: `if (url.startsWith('https://')) window.open(url, '_blank')`. Can be done in a follow-up.
**Status**: SHOULD FIX (non-blocking, defense-in-depth)
